# TransBoard — 病棟移送管理ダッシュボード

> 病棟から検査室への患者移送をリアルタイムで管理する、無料・オープンソースのデスクトップアプリです。

[![License: MIT](https://img.shields.io/badge/License-MIT-blue.svg)](LICENSE)
[![Platform: Windows](https://img.shields.io/badge/Platform-Windows%2010%2F11-blue)](https://github.com/pocket-dev-id/transboard/releases)

## 概要

TransBoard は、病院の看護師と検査室スタッフが**患者の移送状態をリアルタイムで共有**するためのダッシュボードアプリです。

電話やホワイトボードによる状況確認を減らし、「移動中」「検査中」「迎え要」などの状態を全スタッフが一画面で把握できます。追加のサーバーやクラウド契約は不要で、院内のPCにインストールするだけで使い始められます。

> **⚠ 本ソフトウェアは医療機器ではありません。** 詳細は[免責事項](#免責事項)をご確認ください。

---

## 主な機能

- **リアルタイム病床マップ** — 全病床の移送状態を一画面で。5秒ごとに自動更新
- **検査室ビュー** — 検査室側のPCで専用画面を表示。ボタン一つで状態記録
- **通知音・アラート** — 「迎え要」「あと10分」をステータス変化に応じて音声通知
- **電子カルテ CSV 自動取込** — 指定フォルダへのCSV配置で患者情報を自動更新（Shift-JIS/UTF-8 自動判定）
- **複数端末同期（親機・子機モード）** — 同一LAN内の複数PC・タブレットからアクセス可能
- **ICカードスキャン対応** — NFCリーダーで患者IDを自動入力
- **タイムライン・履歴表示** — 1日の移送履歴を時系列で確認
- **院内通話パネル（WebRTC）** — 親機経由のシグナリングによるアプリ間音声通話
- **パスコード保護** — 設定画面をパスコードでロック
- **データバックアップ・復元** — ワンクリックでJSON形式のバックアップを作成

## 移送状態フロー

```
在床 → 出棟登録済 → 移動中 → 検査室到着 → 検査中 → あと10分 → 迎え要 → 帰棟済
                                                                      ↓
                                                               （キャンセル）
```

## インストール

### 方法1: インストーラーを使う（推奨）

1. [Releases ページ](https://github.com/pocket-dev-id/transboard/releases) から最新の `TransBoard-Setup-x.x.x.exe` をダウンロード
2. インストーラーをダブルクリックして実行
3. 画面の指示に従いインストール完了
4. スタートメニューまたはデスクトップの「TransBoard」から起動

### 方法2: ソースから実行（開発者向け）

```bash
# リポジトリをクローン
git clone https://github.com/pocket-dev-id/transboard.git
cd transboard

# 依存パッケージをインストール
npm install

# 開発モードで起動
npm start
```

### ビルド

```bash
# Windowsインストーラーを生成
npm run dist
```

## 初期設定

初回起動時に**セットアップウィザード**が自動的に表示されます。以下の手順で設定してください。

1. **病棟の登録** — 病棟名・内線番号を入力
2. **病床の登録** — 病室番号・病床番号・病床種別（一般/隔離/ICU）を登録
3. **検査室の登録** — CT室・MRI室など検査室の名称・内線番号を登録
4. **検査種別の登録** — CT・MRI・X線など検査の標準所要時間を設定
5. **パスコードの設定** — 設定画面のロック用パスコードを変更（デフォルト: `0000`）

> **セキュリティ:** 初回起動後、必ずパスコードをデフォルトの `0000` から変更してください。

## 複数端末での運用（親機・子機モード）

親機が API サーバー（ポート 3005）を立ち上げ、子機アプリがそこに接続してデータを共有します。

### 親機の設定

1. 設定画面 → 共有設定 → **「親機として動作する」** を選択
2. 親機の IP アドレスが設定画面に表示されます（例: `192.168.1.10`）
3. ファイアウォールで **ポート 3005** を許可してください

### 子機の設定

1. 子機 PC に TransBoard をインストールして起動
2. 設定画面 → 共有設定 → **「子機として動作する」** を選択
3. **親機の IP アドレス**（例: `192.168.1.10`）を入力して保存

> **注意:** 親機・子機は同一ネットワーク（院内 LAN）内に接続されている必要があります。

## 電子カルテ CSV 取込

電子カルテシステムから出力したCSVを自動で取り込む機能です。

### 設定手順

1. 設定画面 → 「CSVインポート」タブ
2. 監視フォルダを指定（UNCパス `\\server\share` も可）
3. カラムマッピングを設定（病床番号・患者ID・患者名の列名を指定）
4. 取込スケジュールを設定（リアルタイム監視 / 定期実行 / 時刻指定）

### 対応CSVフォーマット

| 列 | 設定キー | 説明 |
|----|---------|------|
| 病床番号 | `bed_number` | 病床を特定するキー（必須） |
| 患者ID | `patient_id` | 患者識別番号 |
| 患者氏名 | `patient_name` | 患者の氏名 |
| 在床フラグ | `is_present` | 「在床」「1」「true」などを在床として判定 |

文字コードは **UTF-8（BOMあり/なし）** および **Shift-JIS** を自動判定します。

## データの保存場所

| 設定 | パス |
|------|------|
| ユーザー個別（デフォルト） | `C:\Users\<ユーザー名>\AppData\Roaming\transboard\db.json` |
| 全ユーザー共有 | `C:\ProgramData\transboard\db.json` |

設定画面 → 「詳細設定」からデータ保存先を切り替えられます。

### バックアップ

設定画面 → 「データ管理」→ 「バックアップを作成」でJSONファイルとして書き出せます。

## セキュリティに関する注意

- 本アプリの親機サーバーは院内LANでの使用を前提としています。**インターネットへの公開は行わないでください。**
- デフォルトのパスコード `0000` は初回起動後すぐに変更してください。
- ODBCパスワード・SMBパスワードはOS標準の暗号化機能（Electron `safeStorage`）で保護されます。
- 外部からのHTTPアクセスに対して、パスコード等の機密設定項目へのアクセスは自動的にブロックされます。
