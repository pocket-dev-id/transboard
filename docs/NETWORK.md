# TransBoard ネットワーク要件 (インフラ #5)

## アーキテクチャ概要

```
+------------------+     HTTP (LAN)     +------------------+
|  親機 (Parent)   | <----------------> |  子機 (Client)   |
|  TransBoard App  |    ポート 3005     |  TransBoard App  |
+------------------+                   +------------------+
        |
        | (ローカルアクセス)
        v
+------------------+
|  SMB共有フォルダ  |  (CSV/スケジュール取り込み)
|  ODBCデータソース |  (電子カルテ連携)
+------------------+
```

---

## 使用ポート・プロトコル

| ポート | プロトコル | 方向 | 用途 |
|---|---|---|---|
| **3005** | HTTP (TCP) | 親機→子機 双方向 | REST API（DB同期・デバイスハートビート） |
| **3005** | HTTP (TCP) | 子機→親機 | WebRTCシグナリング（`/webrtc/send`, `/webrtc/poll`） |
| **3005** | HTTP (TCP) | 子機→親機 | アップデートファイル配信（`/updates/`） |
| **動的** | WebRTC/DTLS | P2P | 音声通話（ICE経由のUDP/TCP） |
| **445** | SMB/CIFS | 親機→ファイルサーバー | CSV/スケジュールファイル取り込み |
| **1433等** | ODBC/TCP | 親機→DBサーバー | 電子カルテ連携 (DSN依存) |

---

## ファイアウォール設定（IT部門向け）

### 最小構成（LAN内運用）

1. **親機のインバウンド**: TCP 3005 を **LAN内のみ**で許可
2. **子機のアウトバウンド**: TCP 3005 への送信を **LAN内のみ**で許可
3. **インターネットアクセス**: アプリ自体は**不要**（自動更新機能を使わない場合）

### SMB連携がある場合

4. **親機のアウトバウンド**: TCP 445 をSMBサーバーへ許可

### ODBC連携がある場合

5. **親機のアウトバウンド**: ODBCデータソースのポート（SQL Server: 1433等）を許可

## セキュリティ上の注意点 (セキュリティ #6)

### 現状の暗号化状態

| 通信経路 | 暗号化 | 備考 |
|---|---|---|
| 親機↔子機 REST API | **なし (HTTP)** | LAN内のため短期的リスクは低いが、将来的にHTTPS化を推奨 |
| 親機↔子機 WebRTCシグナリング | **なし (HTTP)** | 同上 |
| WebRTC音声通話 | **あり (DTLS-SRTP)** | WebRTC標準により暗号化済み |
| ODBC接続文字列（保存） | **あり (safeStorage)** | Electron safeStorageで暗号化保存 |
| SMBパスワード（保存） | **あり (safeStorage)** | Electron safeStorageで暗号化保存 |

### HTTPS化の推奨手順（将来対応）

1. 自己署名証明書を親機に生成（`openssl req -x509 ...`）
2. `main.js` の `http.createServer()` を `https.createServer({ key, cert }, ...)` に変更
3. 子機側で証明書を信頼リストに追加（または `rejectUnauthorized: false` で検証スキップ）
4. Electron の `--ignore-certificate-errors` フラグは使用不可（セキュリティ低下）

## 帯域幅・レイテンシ要件

| 項目 | 推奨値 | 備考 |
|---|---|---|
| LAN帯域 | 100Mbps以上 | 通常の有線LANで十分 |
| レイテンシ | 50ms未満 | 有線LANで通常問題なし |
| ポーリング間隔 | 5秒（デフォルト） | `CONFIG.POLL_INTERVAL` で変更可 |
| 接続子機台数 | 制限なし（実運用目安: 〜20台） | 親機のCPU/メモリに依存 |

## ネットワーク診断

子機から親機への疎通確認:
```bash
# 疎通確認（ポート3005）
curl http://<親機IP>:3005/tables/wards

# または Windows PowerShell
Test-NetConnection -ComputerName <親機IP> -Port 3005
```

子機の設定画面「接続テスト」ボタンでもGUIから確認可能。
